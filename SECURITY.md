# Política de Segurança

## Versões com Suporte

Apenas a versão mais recente do plugin recebe correções de segurança.

| Versão | Suportada          |
| ------ | ------------------ |
| 0.1.16 | :white_check_mark: |
| < 0.1.16 | :x:             |

## Reportando uma Vulnerabilidade

Se você descobriu uma vulnerabilidade de segurança neste plugin, **não abra uma issue pública**. Isso evita a exposição do problema antes que uma correção esteja disponível.

### Como reportar

Envie um e-mail para **dead.zl@ifrn.edu.br** com o assunto:

```
[SEGURANÇA] mod_medalhasproitec - <descrição breve>
```

Inclua na mensagem:

- Descrição detalhada da vulnerabilidade
- Passos para reproduzir o problema
- Versão do plugin e versão do Moodle afetados
- Impacto potencial (ex.: acesso não autorizado, vazamento de dados)
- Sugestão de correção, se houver

### O que esperar

| Prazo | Ação |
| ----- | ---- |
| Até 3 dias úteis | Confirmação de recebimento do relatório |
| Até 10 dias úteis | Avaliação inicial e comunicação sobre aceitação ou rejeição |
| Até 30 dias úteis | Disponibilização de uma versão corrigida (quando aplicável) |

Se a vulnerabilidade for **aceita**, você será notificado quando a correção for liberada e receberá crédito pelo relato (a menos que prefira permanecer anônimo).

Se a vulnerabilidade for **rejeitada**, forneceremos uma justificativa clara.

### Divulgação responsável

Pedimos que você mantenha o sigilo sobre a vulnerabilidade até que uma correção seja publicada. Após a correção, a divulgação coordenada pode ser realizada de acordo com combinação entre as partes.

## Escopo

Esta política cobre exclusivamente o plugin **mod_medalhasproitec**. Vulnerabilidades na plataforma Moodle ou em outros plugins devem ser reportadas aos seus respectivos mantenedores:

- Moodle: <https://moodle.org/security/>
